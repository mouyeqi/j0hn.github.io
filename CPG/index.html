Modeling and Discovering Vulnerabilities with Code Property Graph 笔记

发表于2023-03-14|更新于2023-03-14|论文笔记
|字数总计:1.1k|阅读时长:3分钟|阅读量:

some words

出版时间：2014

简介

静态分析是漏洞分析的一个非常有用的办法，在大规模漏洞分析上尤其有效（动态分析搞大规模需要非常多的算力）。本文承接了过去的静态分析方法，提出了一种新的程序表示方法，并用图遍历的办法来实现优雅的漏洞静态漏洞分析。

贡献

1、提出了CPG（code property graph）来表示源码，结合了抽象语法树、控制流图和程序依赖图

2、证明了常见漏洞可以被表示为CPG上的某种遍历，可以高效而优雅地检测常见漏洞

3、使用成熟的图数据库来实现，使得算法在大规模的代码上也能高效运行

代码的图表示

经典的代码图表示方式有三种：抽象语法树、控制流图、程序依赖图

抽象语法树（AST）

在代码的编译中非常重要的表示方法。将代码转化为一棵树，叶节点是操作数，非叶节点是操作符。
缺陷：AST很适合做简单的代码分析，但由于没有控制流和数据依赖信息，无法分析不可进入的代码和未初始化的代码

控制流图（CFG）

逆向分析工具中的标准表示方法。把代码表示为一个有向图，每个节点都是一块无跳转的执行语句，边是转移条件。能够有效表示程序控制流信息。
缺陷：无法表述数据流

程序依赖图（PDG）

在代码切片中经常使用的表示方法，代码为节点，边是代码之间的控制依赖和数据依赖。显示表示声明和判断之间的关系
缺陷：在生成图的时候打乱了语句顺序，语句执行顺序无法正确推断

代码属性图（CPG）

鉴于经典代码表示方法的不足，论文选择了将这三种方法通过属性图结合在一起，起到互补的作用。

对AST的转换

由于三种方法中AST是唯一一个对代码的语法结构做了细致分析的，结合图的转化第一步是将AST转化为属性图，节点和边都对应转换过去，边标记为AST边，每个节点被赋予属性code和order

对CFG的转换

第二部是融合CFG的信息。这一步中，code属性为STMT和PRED的节点将成为CFG节点的映射，通过CFG中的边关系连接在一起。边被标记上true、false和通配符

对PDG的转换

PDG的节点与CFG的一样，区别在于属性和边。PDG的边为控制和数据依赖，标签也被设置为{C（控制依赖）,D（数据依赖）}。数据依赖边被加上一个属性Symbol，控制依赖边被加上属性true或false。
